# 404 / 50x ページとヘルスチェックの実装パターン

Next.js(App Router)での実装。

## 404 / 50x ページ

- not-found.tsx を root に置くと未マッチ URL 全体を受ける。root layout 内に描画されるため共通ヘッダーが残る。主要ページへの戻りリンクを付ける
- error.tsx が通常の 50x を受ける。"use client" 必須。エラートラッキングへの captureException と reset ボタンを付ける
- global-error.tsx は root layout ごと壊れたときだけ出る最後の砦。error.tsx と両方置き、文言のトーンを揃える
- 認証ミドルウェアが未ログインを一律リダイレクトする構成では、404 を見るのは主にログイン済みユーザー。未認証の未知パスはログインページへ行く仕様になることを認識して選ぶ
- error.tsx の自動テストは throw する本番ルートが必要になり難しい。throw する一時ページ + 一時 spec で表示を検証してから消し、404 とヘルスチェックだけを恒久テストにするのが現実的

## ヘルスチェック

- GET /api/health が 200 と固定ボディ(`{"status":"ok"}`)を返す。認証と DB 接続に依存させず、プロセスの生存確認に徹する。DB の死活まで見るかは監視設計で別途判断する
- cache-control: no-store を明示する。CDN・中間キャッシュが応答を保存すると死活を映さなくなる
- 認証ミドルウェアが /api を素通しする構成か確認する。外形監視は未認証で叩く
- E2E は未認証の request.get で 200 とボディを確認する
