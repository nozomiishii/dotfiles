# /src

直近の会話で話していたトピックの一次情報を集めて提案する。出力はリンク集であって解説ではない（解説がほしいときは `/q`）。

## 対象トピックの特定

直近の話題を主対象にする。`$ARGUMENTS` があれば、それで絞り込み・補足する。

特定したトピックを冒頭に 1 行で明示する。

```
対象トピック: ×××
```

## 検索

WebSearch でトピックの一次情報とわかりやすい解説を探す。公式 URL が確実でないときは WebFetch で実在と内容を確認してから載せる。

## 出力

2 セクションに分けて出す。各リンクに「何がわかるか」を一言添える。

### 公式ドキュメント

英語原典を主にする。公式に日本語版があれば併記してよい。

### わかりやすい日本語ブログ

新しく、丁寧で、図解や具体例があるものを優先する。Zenn / Qiita / 企業 tech ブログ / 個人ブログを広く対象にする。古い記事・薄い転載記事は避ける。

## ルール

- 数は固定しない。質の高いものだけを必要なだけ挙げる
- リンクは Markdown のハイパーリンクにする
