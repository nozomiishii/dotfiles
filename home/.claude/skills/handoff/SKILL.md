# /handoff

現在の会話を要約した引き継ぎドキュメントを書き、新しい agent がそのまま作業を続けられるようにする。

- 保存先は **OS の一時ディレクトリ**（`$TMPDIR`、無ければ `/tmp`）。作業中のリポジトリには置かない。
- ドキュメントに「推奨スキル」セクションを含め、次の agent が呼ぶべき skill を挙げる。
- PRD・プラン・ADR・issue・コミット・diff など、既に他の成果物に記録済みの内容は重複させず、パス や URL で参照する。
- API キー・パスワード・個人情報などの機微情報は伏せる。
- 引数が渡された場合は、次のセッションが何に集中するかの説明とみなし、それに合わせてドキュメントを調整する。
