# Learn

技術的なトピックについて最新の情報を調べ、わかりやすく解説する。

## Workflow

### Phase 1: 情報収集

`$ARGUMENTS` のトピックについて最新情報を集める。

1. WebSearch でまず 2-3 回検索する。同じキーワードを繰り返さず、異なるアングルから検索する
   - 例: 概念の定義、実践的な使い方、批判や議論
2. 検索結果から信頼性の高いソースを選ぶ
   - 優先: 公式ドキュメント、著名な技術ブログ、カンファレンストーク、GitHub Discussion
   - 避ける: SEO 記事、内容の薄いまとめ記事
3. 重要なページを WebFetch で取得する
4. 情報が不足していると感じたら、追加で検索・取得する。回数を固定せず、トピックの複雑さに応じて柔軟に判断する

### Phase 2: 解説

収集した情報を踏まえて、以下の構造で解説する。

#### 解説の構造

- **一言でいうと**: トピックの核心を 1-2 文で伝える
- **背景**: なぜこの概念が生まれたか、どんな問題を解決するか
- **詳しい説明**: アナロジーや具体例を交えて噛み砕いて伝える
- **実践での活用**: 実際にどう使われているか、何が変わるか

#### 出典

解説の最後に、参照した URL をリストで明示する。各ソースから何を得たかを簡潔に添える。

```
## Sources
- [タイトル](URL) — この記事から○○について得た
- [タイトル](URL) — ○○の具体例として参照
```

### Phase 3: 議論の余白を残す

解説して終わりではなく、会話が自然に続く余韻を残す。

- 調べた中で特に興味深かった点や、ソース間で意見が分かれていた点を軽く添える
- 明示的な理解度確認（「わかりましたか？」「ここまで大丈夫ですか？」）はしない
- ユーザーが追加で質問してきたら、結論を急がず一緒に考える姿勢で掘り下げる

## Rules

- **調べた情報のみで回答する**: 古い学習データで埋めない。WebSearch/WebFetch で確認できた情報に基づいて回答する。確認できていない情報を補足する場合は、その旨を明示する
- **情報の羅列にしない**: 噛み砕いて、ユーザーが理解できる言葉で説明する
- **出典なしの主張をしない**: どのソースからの情報か明示する
- **日本語で解説する**: 技術用語やコード例はそのまま使ってよい
