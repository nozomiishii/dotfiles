# グローバル指示

## 言語
- 日本語で応答する。技術用語はそのまま使用してよい。

## コミュニケーション
- 質問は1つずつする。
- 推測せず、実装と公式ドキュメントを必ず確認する
- 返答でGitHubのIssueやPRに言及する時は、必ずリンクにする。

## 実装
- 設計ドキュメント・brainstormingの成果はGitHub issueにまとめる。
- コメントは、短くわかりやすい日本語で書く。指摘されたら、必要な分だけ伸ばす。

## 検索
- GitHub IssueはClosedやduplicatedになっている場合、Openしているissueまで特定し提案する。

## 用語
- Routines: Claude Code の Routines 機能 <https://code.claude.com/docs/ja/routines>
